# Compatibility

## Authority

This document is authoritative for language compatibility direction and overclaim controls.

## Capability Direction

Deep capability goals:

- TypeScript
- JavaScript
- Python

Structural capability goals:

- Go
- Rust
- Java
- C#
- Kotlin

Generic graceful fallback goals:

- C
- C++
- Ruby
- PHP
- Swift
- Dart
- Shell
- Lua
- Elixir
- Scala
- unknown text-based languages

## Capability Honesty

Proofrail MUST expose adapter capability negotiation. Unknown or partially supported languages must degrade honestly.

Do not use marketing language such as broad "full support" unless the exact capability set supports that claim.

## Current Phase

No language adapter is implemented in Phase 0. Compatibility direction is documentation only.
