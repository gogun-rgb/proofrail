# Non-Goals

## Authority

This document is authoritative for what Proofrail must not become.

## Product Non-Goals

Proofrail is not:

- an AI code reviewer
- a chat application
- a code quality scoring tool
- an AI confidence system
- an LLM-based approval engine
- a replacement for deterministic CI checks
- a general repository intelligence dashboard
- a human taste or style adjudicator

## Phase 0 Non-Goals

Phase 0 MUST NOT implement:

- Proofrail runtime behavior
- product kernel logic
- repository inspection
- semantic change observation
- verification execution
- policy evaluation
- language adapters
- CLI, API, server, MCP server, web UI, or GitHub integration
- production package placeholders that imply those systems exist

## Overclaim Controls

Proofrail MUST NOT claim broad language support unless adapter capability negotiation describes the exact capability level.

Proofrail MUST NOT use phrases such as "AI approved", "high confidence", or "probably safe" as authoritative outcome language.

Proofrail MUST NOT let natural-language summaries substitute for evidence, verification receipts, policies, rules, or deterministic verdict derivation.
