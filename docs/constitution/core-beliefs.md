# Core Beliefs

## Authority

This document is authoritative for Proofrail product philosophy. It does not define protocol syntax or implementation behavior.

## Beliefs

1. Claim is not evidence. Verify it.
2. Deterministic observations and verification receipts are stronger than persuasive explanations.
3. AI agents can produce valuable changes, but they must not authorize their own admissibility.
4. Machine clients need stable schemas, reason codes, and remediation-relevant context before they need prose.
5. Human operators need fast operational comprehension, not confidence theater.
6. Trust boundaries should be explicit enough for future agents to preserve without oral history.
7. Unsupported or degraded capability must be represented honestly.
8. Evidence history should be append-only in concept; later authoritative decisions may supersede earlier ones without rewriting them.

## Design Consequence

Proofrail prefers traceable deterministic inputs over natural-language assertions. Natural language may explain an outcome, but it MUST NOT change authoritative verdict semantics.
